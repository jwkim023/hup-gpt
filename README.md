# [Project Overview] Spotfire 지능형 데이터 허브 및 생성형 AI 추천 시스템

## 1. 프로젝트 배경 및 목적

* **배경:** 사내에 파편화된 제조 데이터를 통합하고, 현업 사용자가 복잡한 쿼리 없이 필요한 데이터를 즉시 찾을 수 있는 환경 필요.
* **목적:** Spotfire를 단일 창구(Single Source of Truth)로 활용하여 데이터 공급 체계를 자동화하고, LLM을 통해 사용자 의도에 맞는 데이터를 추천 및 시각화함.

---

## 2. 핵심 기술 스택

* **Frontend:** TIBCO Spotfire Text Area (HTML5, CSS3, JavaScript)
* **Interface:** Spotfire Data Function (Python) & Document Properties
* **Backend LLM:** **gpt-oss-20b** (MoE 아키텍처, 활성 파라미터 3.6B)
* *특징:* o3-mini급 추론 성능, 저사양 VRAM(16GB+) 환경 최적화


* **Data Protocol:** **MCP (Model Context Protocol)**
* *역할:* DB 메타데이터(Schema)를 모델에 효율적으로 주입하여 데이터 매핑 정확도 향상



---

## 3. 시스템 아키텍처 (Workflow)

1. **사용자 입력:** Spotfire 텍스트 영역 내 구현된 채팅창(JS)을 통해 자연어 질의 입력.
2. **질의 전달:** JavaScript가 입력값을 문서 속성에 저장하고, Python 스크립트가 LLM 서버로 전달.
3. **메타데이터 조회:** LLM이 질문을 분석하고, **MCP 서버**를 통해 관련 테이블/컬럼 정보(Metadata) 탐색.
4. **데이터 추천:** LLM이 최적의 데이터 테이블을 추천하거나 직접 SQL을 생성하여 결과 응답.
5. **동적 시각화:** 사용자가 승인 시, Spotfire가 해당 데이터를 즉시 로드하고 대시보드 업데이트.

---

## 4. 모델 선정 사유: gpt-oss-20b

* **고효율 추론:** MoE(Mixture of Experts) 구조로 전체 20B 크기 대비 매우 빠른 응답 속도 보장.
* **경제성:** Unsloth 최적화 시 13~16GB VRAM에서 구동 가능하여 사내 워크스테이션급 서버로 운영 가능.
* **논리력:** 데이터 스키마 간의 복잡한 관계를 파악하고 사용자의 모호한 질문을 구체적인 DB 엔티티로 치환 가능.

---

## 5. 기대 효과 및 향후 계획

* **기대 효과:**
* IT 담당자의 반복적인 데이터 추출 업무 80% 이상 감소.
* 현업 사용자의 데이터 접근성 및 셀프 서비스 분석 역량 강화.
* 사내 고유 데이터 보안 유지 (로컬 LLM 및 MCP 활용으로 외부 유출 차단).


* **향후 계획 (2026 로드맵 연계):**
* 자산관리 시스템 데이터와 연동하여 실시간 자산 실사 분석 대시보드 구현.
* 해당 프로젝트 경험을 바탕으로 **정보관리기술사** 실무 사례(D.X, 스마트 팩토리)로 자산화.
